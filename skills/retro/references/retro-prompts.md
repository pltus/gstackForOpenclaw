# Retro prompts

Use these prompts when the retrospective needs more depth.

## Outcome framing

- What did the team expect to happen?
- What actually happened?
- Which outcomes mattered most for users, operators, or the team?

## Success patterns

- Which decisions reduced risk or increased speed?
- What should become a standard practice next time?
- Which part of the plan was sharper than usual, and why?

## Failure patterns

- Where did the team discover problems too late?
- Which assumptions were never really validated?
- What handoff, ownership, or sequencing gap caused the most drag?
- Which issue looked like execution failure but was actually planning failure?

## Action quality

- What is the smallest change that would prevent this class of problem next time?
- Which action item has a clear owner and trigger?
- Which proposed fix is too vague or too big to survive?
- What should be stopped, not merely improved?
