# Office-hours routing guide

Use this guide when the request is broad and you need to decide which deeper skill should take over.

- Use `plan-ceo-review` when the main question is product wedge, ambition, user value, or scope shape.
- Use `plan-eng-review` when the main question is feasibility, architecture, rollout, or test strategy.
- Use `review` when there is already a concrete diff, PR, or implementation to inspect.
- Use `investigate` when the system is already behaving badly and the root cause is unclear.
- Use `retro` when the work already happened and the team needs lessons, patterns, or process changes.

If none fit cleanly, keep `office-hours` as a short framing and decision aid rather than inventing a heavier process.
